Factory Inventory Management System

A demo full-stack application for tracking inventory, orders, demand forecasts, backlog, and spending across a multi-warehouse operation. Vue 3 SPA on the front, FastAPI on the back, JSON files as the data store.

System Architecture

Data Flow — Request Lifecycle

1

User adjusts a filter

FilterBar.vue updates shared refs in the useFilters composable (warehouse, category, status, month).

2

View reacts to filter change

A view (e.g. Dashboard.vue) watches the filters and calls the matching method on api.js.

3

Axios HTTP request

api.js builds URLSearchParams, skipping any filter set to "all", and GETs http://localhost:8001/api/....

4

FastAPI route receives request

CORS middleware allows cross-origin. The route accepts query params as Optional[str] and delegates to filter helpers.

5

In-memory filtering

apply_filters and filter_by_month in main.py iterate the in-memory list and return a filtered copy.

6

Pydantic validates response

The response_model serializes results through InventoryItem, Order, etc. — guaranteeing shape.

7

Vue stores raw data in refs

Raw arrays land in refs like allOrders and inventoryItems. Derived values live in computed().

8

Template renders

Tables, custom-SVG charts, and CSS-Grid dashboards re-render. Modals open per-row for detail views.
